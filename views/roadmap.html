Roadmap

This roadmap contains everything that's planned for mid-December.

Got proposal for the next release?

Feather project will concentrate on delivering stock Sitefinity dynamic module widget based on the MVC/Razor technology providing automatically generated clean markup templates. The widget will be open source and will also be built with reusable elements that can be easily used in other custom widgets.

The stock dynamic module widget will come with the following functionality:

  • Automatically generated razor widget templates based on the dynamic content model
  • Supports master/detail mode with separate razor templates
  • Support for hierarchical modules (design time and run time filtering(url based) by parent item)
  • Supports filtering by taxonomies (design time and run time through url)
  • Supports custom list mode (show a user picked (in design time) set of items in a list)
  • Support filtering by date range
  • Support sorting and paging
  • We will also provide well documented infrastructure for rapid building of custom widgets that have similar functionality to the stock dynamic module widget.

The reusable elements that we are going to provide gradually until mid-December are:

  • Content Selectors that can be used in Feather widget designers or any angular based UI in Sitefinity
    • Dynamic module content selector
    • Pages selector
    • Hierarchical taxonomy selector (categories) (single and multi - choice)
    • Custom taxonomy selector (auto populated with all taxonomies used in a type including custom field taxonomies)
    • All selectors will allow for multi-choice - e.g. can select more than one content item
  • MVC Partial views and helpers
    • For showing media fields
    • For showing related data fields
    • For showing hierarchical data

We will provide a public open source sample web site

The website will show case the use of all progress on the Feather project. The site will be made with mobile responsive design and will show case sample usages of the widgets that utilize features like hierarchical modules, related content, inline editing, canonical locations.
